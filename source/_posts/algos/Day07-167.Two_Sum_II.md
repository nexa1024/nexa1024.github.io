## 题目概述
**双指针经典应用**。本题是 LeetCode 第 167 题，核心考点是**利用数组已排序的特性，使用双指针在对撞查找中找到目标和的两个元素**。

## 为什么这道题重要
这是 **双指针对撞查找** 的经典应用，是 LeetCode 必考题。本题与"1. 两数之和"的区别在于：

### 与两数之和（No.1）的对比
| 题目 | 数组状态 | 返回值 | 最优解法 | 空间复杂度 |
|------|---------|--------|---------|-----------|
| 1. 两数之和 | **无序** | 下标 | HashMap | O(n) |
| **167. 两数之和 II** | **已排序** | 下标（从1开始） | **双指针** | **O(1)** |

### 解法延伸路径
- **双指针对撞法（本题推荐）** → 延伸至：
  - [15. 三数之和](https://leetcode.cn/problems/3sum/)（固定一个数 + 双指针）
  - [18. 四数之和](https://leetcode.cn/problems/4sum/)（固定两个数 + 双指针）
  - [11. 盛最多水的容器](https://leetcode.cn/problems/container-with-most-water/)（面积优化问题）

⚠️ **关键区别**：本题要求**常量级空间复杂度 O(1)**，因此不能使用 HashMap。由于数组已排序，双指针法是最优解。

## 解法对比
| 解法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| 暴力枚举 | O(n²) | O(1) | 性能差，不推荐 |
| **二分查找** | **O(n log n)** | **O(1)** | 可行但非最优 |
| **双指针对撞** | **O(n)** | **O(1)** | **本题最优解，利用排序特性** |
| HashMap | O(n) | O(n) | 不符合题目常量空间要求 |

**背诵重点**：双指针对撞解法（O(n) 时间，O(1) 空间）

### 为什么双指针最优？
1. **数组已排序**：排序是有序信息，双指针法利用这一特性
2. **常量空间**：只需两个指针变量，空间复杂度 O(1)
3. **线性时间**：每次迭代至少排除一个元素，最多 n-1 次迭代

## 题目
给你一个下标从 **1** 开始的整数数组 `numbers`，该数组已按**非递减顺序排列**，请你从数组中找出满足相加之和等于目标数 `target` 的两个数。如果设这两个数分别是 `numbers[index1]` 和 `numbers[index2]`，则 `1 <= index1 < index2 <= numbers.length`。

以长度为 2 的整数数组 `[index1, index2]` 的形式返回这两个整数的下标 `index1` 和 `index2`。

你可以假设每个输入**只对应唯一的答案**，而且你**不可以**重复使用相同的元素。

你所设计的解决方案必须只使用**常量级的额外空间**。

**示例 1：**
```
输入：numbers = [2,7,11,15], target = 9
输出：[1,2]
解释：2 与 7 之和等于目标数 9 。因此 index1 = 1, index2 = 2 。返回 [1, 2] 。
```

**示例 2：**
```
输入：numbers = [2,3,4], target = 6
输出：[1,3]
解释：2 与 4 之和等于目标数 6 。因此 index1 = 1, index2 = 3 。返回 [1, 3] 。
```

**示例 3：**
```
输入：numbers = [-1,0], target = -1
输出：[1,2]
解释：-1 与 0 之和等于目标数 -1 。因此 index1 = 1, index2 = 2 。返回 [1, 2] 。
```

**提示：**
- `2 <= numbers.length <= 3 * 10⁴`
- `-1000 <= numbers[i] <= 1000`
- `numbers` 按**非递减顺序**排列
- `-1000 <= target <= 1000`
- **仅存在一个有效答案**

## 标签
数组、双指针、二分查找

## 背诵模型
**模型/状态**

+ 使用**两个指针**：`left` 指向数组开头（索引 0），`right` 指向数组末尾（索引 n-1）
+ 目标：找到 `numbers[left] + numbers[right] == target`

**不变量/约束**

+ `left` 和 `right` 始终指向**有效范围内的下标**：`0 <= left < right < n`
+ 答案**一定存在**且**唯一**（题目保证）
+ 每次迭代后，搜索范围**至少减少一个元素**

**推进/转移**

+ 计算 `sum = numbers[left] + numbers[right]`
+ **三种情况**：
  1. `sum == target`：找到答案，返回 `[left + 1, right + 1]`（注意下标从 1 开始）
  2. `sum < target`：说明和太小，需要增大 → `left++`（向右移动左指针，选更大的数）
  3. `sum > target`：说明和太大，需要减小 → `right--`（向左移动右指针，选更小的数）

**核心逻辑图解**
```
numbers = [2, 7, 11, 15], target = 9

初始状态：
[2,  7, 11, 15]
 ↑          ↑
left       right
sum = 2 + 15 = 17 > 9 → right--

第一次移动：
[2,  7, 11, 15]
 ↑      ↑
left   right
sum = 2 + 11 = 13 > 9 → right--

第二次移动：
[2,  7, 11, 15]
 ↑  ↑
left right
sum = 2 + 7 = 9 == target → 返回 [1, 2]
```

**为什么这个策略正确？**

由于数组**已排序（非递减）**：
- 当 `sum < target` 时，`left` 当前的数太小，任何比它更小的数（在 left 左侧）都不可能满足条件
- 当 `sum > target` 时，`right` 当前的数太大，任何比它更大的数（在 right 右侧）都不可能满足条件
- 因此可以**安全地排除**不可能的元素

**终止/答案+边界**

+ **正常终止**：找到 `sum == target`，返回 `[left + 1, right + 1]`
+ **边界情况**：
  - 题目保证**有且仅有**一个有效答案，因此不需要处理"无解"情况
  - 最小数组长度为 2，此时 left = 0, right = 1

**复杂度**

+ 时间复杂度：**O(n)**，其中 n 为 numbers 的长度。每次迭代 left 增加或 right 减少，最多迭代 n-1 次
+ 空间复杂度：**O(1)**，只使用两个指针变量

**常见坑**

+ **下标从 1 开始**：返回时需要 `left + 1` 和 `right + 1`
+ **不要使用 HashMap**：题目要求常量级空间复杂度 O(1)
+ **整数溢出**：虽然本题数据范围小（-1000 到 1000），但实际面试中要注意 `sum` 可能溢出，使用 `long` 或提前判断

## 代码
### java
```plain
class Solution {
    public int[] twoSum(int[] numbers, int target) {
        int left = 0;                           // 左指针，指向数组开头
        int right = numbers.length - 1;         // 右指针，指向数组末尾

        while (left < right) {
            int sum = numbers[left] + numbers[right];

            if (sum == target) {
                // 找到答案，注意下标从 1 开始
                return new int[]{left + 1, right + 1};
            } else if (sum < target) {
                // 和太小，需要更大的数，左指针右移
                left++;
            } else {
                // 和太大，需要更小的数，右指针左移
                right--;
            }
        }

        // 题目保证有解，这行代码理论上不会执行
        return new int[]{-1, -1};
    }
}
```

## 方法二：二分查找法（非最优但值得一提）

虽然双指针是最优解，但二分查找法也值得一提：

```java
class Solution {
    public int[] twoSum(int[] numbers, int target) {
        for (int i = 0; i < numbers.length; i++) {
            int complement = target - numbers[i];
            // 在 [i+1, n-1] 范围内二分查找 complement
            int left = i + 1, right = numbers.length - 1;
            while (left <= right) {
                int mid = left + (right - left) / 2;
                if (numbers[mid] == complement) {
                    return new int[]{i + 1, mid + 1};
                } else if (numbers[mid] < complement) {
                    left = mid + 1;
                } else {
                    right = mid - 1;
                }
            }
        }
        return new int[]{-1, -1};
    }
}
```

**复杂度分析：**
- 时间复杂度：O(n log n)，外层循环 n 次，每次二分查找 O(log n)
- 空间复杂度：O(1)

**为什么不推荐？**
- 双指针法已经是 O(n)，比二分查找更优
- 代码更简洁，思路更直观
