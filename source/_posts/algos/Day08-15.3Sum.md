## 题目概述
**排序+双指针经典应用**。本题是 LeetCode 第 15 题，核心考点是**先排序，再固定一个数，使用双指针查找另外两个数，同时去重**，是面试高频题。

## 为什么这道题重要
这是 **双指针查找 + 去重技巧** 的经典应用，是 LeetCode 必考题。本题与"1. 两数之和"的区别在于：

### 与两数之和（No.1）的对比
| 题目 | 数组状态 | 返回值 | 最优解法 | 核心难点 |
|------|---------|--------|---------|---------|
| 1. 两数之和 | 无序 | 下标 | HashMap | 无 |
| **15. 三数之和** | **可排序** | **元素值组合** | **排序+双指针** | **去重** |

### 解法延伸路径
- **排序 + 双指针法（本题推荐）** → 延伸至：
  - [18. 四数之和](https://leetcode.cn/problems/4sum/)（固定两个数 + 双指针）
  - [16. 最接近的三数之和](https://leetcode.cn/problems/3sum-closest/)（类似思路，找最小差值）

⚠️ **关键区别**：本题要求返回所有**不重复**的三元组，且三元组内的元素值之和为 0。哈希法去重复杂，双指针法配合排序是最优解。

## 解法对比
| 解法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| 暴力枚举 + Set去重 | O(n³) | O(n) | 性能差，不推荐 |
| **排序 + 双指针** | **O(n²)** | **O(1)** | **最优解，利用排序特性+双指针去重** |
| HashMap + 去重 | O(n²) | O(n) | 去重逻辑复杂，不推荐 |

**背诵重点**：排序 + 双指针解法（O(n²) 时间，O(1) 空间，去重是关键）

### 为什么双指针最优？
1. **排序后有序**：可以双指针对撞查找，避免三层循环
2. **去重容易**：排序后相同元素相邻，去重逻辑清晰
3. **常量空间**：除排序外（可不计入空间复杂度），只需指针变量

## 题目
给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。请你返回所有和为 `0` 且不重复的三元组。

**注意**：答案中不可以包含重复的三元组。

**示例 1：**
```
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0。
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0。
不同的三元组是 [-1,0,1] 和 [-1,-1,2]。
注意，输出的顺序和三元组的顺序并不重要。
```

**示例 2：**
```
输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0。
```

**示例 3：**
```
输入：nums = [0,0,0]
输出：[[0,0,0]]
解释：唯一可能的三元组和为 0。
```

**提示：**
- `3 <= nums.length <= 3000`
- `-10⁵ <= nums[i] <= 10⁵`

## 标签
数组、双指针、排序

## 背诵模型
**模型/状态**

+ **先排序**：将数组按升序排序，这是双指针法的前提
+ **外层循环**：固定第一个数 `nums[i]`（从 0 到 n-3）
+ **内层双指针**：`left = i + 1`，`right = n - 1`，寻找 `nums[i] + nums[left] + nums[right] == 0`

**不变量/约束**

+ `i` 在 `[0, n-3]` 范围内
+ `left` 和 `right` 满足 `i < left < right < n`
+ 数组**已排序**，保证了双指针法的正确性

**推进/转移**

+ **外层循环**：遍历每个可能的第一个数 `nums[i]`
  - **去重①**：如果 `i > 0` 且 `nums[i] == nums[i-1]`，跳过（避免第一个数重复）

+ **内层双指针查找**：计算 `sum = nums[i] + nums[left] + nums[right]`
  1. **sum == 0**：找到一个有效三元组
     - 记录结果：`[nums[i], nums[left], nums[right]]`
     - **去重②**：`left` 右移跳过重复元素 `while left < right && nums[left] == nums[left+1]: left++`
     - **去重③**：`right` 左移跳过重复元素 `while left < right && nums[right] == nums[right-1]: right--`
     - 同时移动 `left++` 和 `right--` 继续查找
  2. **sum < 0**：说明和太小，需要增大 → `left++`（向右移动左指针，选更大的数）
  3. **sum > 0**：说明和太大，需要减小 → `right--`（向左移动右指针，选更小的数）

**核心逻辑图解**
```
nums = [-1, 0, 1, 2, -1, -4]
排序后：[-4, -1, -1, 0, 1, 2]

第一轮 i=0, nums[i]=-4:
[-4, -1, -1, 0, 1, 2]
  ↑i      ↑left    ↑right
sum = -4 + (-1) + 2 = -3 < 0 → left++
继续直到 sum > 0，无解

第二轮 i=1, nums[i]=-1:
[-4, -1, -1, 0, 1, 2]
       ↑i  ↑left       ↑right
sum = -1 + (-1) + 2 = 0 → 找到 [-1, -1, 2]
去重后 left++, right--
[-4, -1, -1, 0, 1, 2]
       ↑i        ↑left ↑right
sum = -1 + 0 + 1 = 0 → 找到 [-1, 0, 1]
去重后 left++, right-- → left >= right，退出

第三轮 i=2, nums[i]=-1:
[-4, -1, -1, 0, 1, 2]
          ↑i ↑left    ↑right
去重①：nums[2] == nums[1] → 跳过
```

**为什么这个策略正确？**

由于数组**已排序（升序）**：
- 固定 `nums[i]` 后，问题转化为"在 `[i+1, n-1]` 范围内找两数之和等于 `-nums[i]`"
- 双指针对撞查找保证 O(n) 时间找到所有组合
- 排序使得相同元素相邻，去重只需跳过相邻的相同元素

**终止/答案+边界**

+ **正常终止**：外层循环遍历完所有可能的第一个数
+ **边界情况**：
  - 数组长度小于 3：返回空列表
  - 所有元素相同且不为 0：返回空列表
  - 所有元素都是 0：返回 `[[0, 0, 0]]`

**复杂度**

+ 时间复杂度：**O(n²)**
  - 排序：O(n log n)
  - 外层循环：O(n)
  - 内层双指针：O(n)
  - 总复杂度：O(n log n) + O(n²) = **O(n²)**
+ 空间复杂度：**O(1)**（不考虑排序的栈空间）

**常见坑**

+ **去重是核心难点**：
  1. 第一个数去重：`if i > 0 && nums[i] == nums[i-1]: continue`
  2. 第二个数去重：找到答案后 `left` 跳过重复元素
  3. 第三个数去重：找到答案后 `right` 跳过重复元素
+ **不要使用 HashMap**：去重逻辑复杂，且空间复杂度 O(n)
+ **注意边界**：数组长度、指针移动条件、去重判断的顺序


## 代码
### java
```plain
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();

        // 1. 先排序
        Arrays.sort(nums);

        // 2. 外层循环：固定第一个数
        for (int i = 0; i < nums.length - 2; i++) {
            // 去重①：如果第一个数重复，跳过
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }

            // 优化：如果最小的三个数之和都大于0，后面不可能有解
            if (nums[i] + nums[i + 1] + nums[i + 2] > 0) {
                break;
            }

            // 优化：如果当前数加上最大的两个数都小于0，当前数太小，跳过
            if (nums[i] + nums[nums.length - 1] + nums[nums.length - 2] < 0) {
                continue;
            }

            // 3. 双指针查找另外两个数
            int left = i + 1;
            int right = nums.length - 1;

            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];

                if (sum == 0) {
                    // 找到一个有效三元组
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));

                    // 去重②：跳过第二个数的重复值
                    while (left < right && nums[left] == nums[left + 1]) {
                        left++;
                    }
                    // 去重③：跳过第三个数的重复值
                    while (left < right && nums[right] == nums[right - 1]) {
                        right--;
                    }

                    // 同时移动指针，继续查找
                    left++;
                    right--;
                } else if (sum < 0) {
                    // 和太小，需要更大的数，左指针右移
                    left++;
                } else {
                    // 和太大，需要更小的数，右指针左移
                    right--;
                }
            }
        }

        return result;
    }
}
```

class Solution {
    fun threeSum(nums: IntArray): List<List<Int>> {
        val result = mutableListOf<List<Int>>()

        // 1. 先排序
        nums.sort()

        // 2. 外层循环：固定第一个数
        for (i in 0 until nums.size - 2) {
            // 去重①：如果第一个数重复，跳过
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue
            }

            // 优化：如果最小的三个数之和都大于0，后面不可能有解
            if (nums[i] + nums[i + 1] + nums[i + 2] > 0) {
                break
            }

            // 优化：如果当前数加上最大的两个数都小于0，当前数太小，跳过
            if (nums[i] + nums[nums.size - 1] + nums[nums.size - 2] < 0) {
                continue
            }

            // 3. 双指针查找另外两个数
            var left = i + 1
            var right = nums.size - 1

            while (left < right) {
                val sum = nums[i] + nums[left] + nums[right]

                when {
                    sum == 0 -> {
                        // 找到一个有效三元组
                        result.add(listOf(nums[i], nums[left], nums[right]))

                        // 去重②：跳过第二个数的重复值
                        while (left < right && nums[left] == nums[left + 1]) {
                            left++
                        }
                        // 去重③：跳过第三个数的重复值
                        while (left < right && nums[right] == nums[right - 1]) {
                            right--
                        }

                        // 同时移动指针，继续查找
                        left++
                        right--
                    }
                    sum < 0 -> {
                        // 和太小，需要更大的数，左指针右移
                        left++
                    }
                    else -> {
                        // 和太大，需要更小的数，右指针左移
                        right--
                    }
                }
            }
        }

        return result
    }
}

## 去重逻辑详解

去重是本题的核心难点，需要从三个层面去重：

### 1. 第一个数去重
```java
// 如果当前第一个数和上一个相同，跳过
if (i > 0 && nums[i] == nums[i - 1]) {
    continue;
}
```

### 2. 第二个数去重
```java
// 找到一个答案后，跳过所有相同的 left 值
while (left < right && nums[left] == nums[left + 1]) {
    left++;
}
```

### 3. 第三个数去重
```java
// 找到一个答案后，跳过所有相同的 right 值
while (left < right && nums[right] == nums[right - 1]) {
    right--;
}
```

### 去重示例
```
nums = [-2, 0, 0, 2, 2]
排序后：[-2, 0, 0, 2, 2]

i=0, nums[i]=-2:
left=1, right=4: sum = -2 + 0 + 2 = 0 → 找到 [-2, 0, 2]
去重后 left=2(跳过0), right=3(跳过2)
left=3, right=3 → 退出内层循环

结果：[[-2, 0, 2]]（只找到一次，没有重复）
```

## 方法二：HashMap 解法（不推荐）

虽然可以使用 HashMap，但去重逻辑复杂，且空间复杂度 O(n)：

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        Set<List<Integer>> resultSet = new HashSet<>();
        Arrays.sort(nums);

        for (int i = 0; i < nums.length - 2; i++) {
            Set<Integer> seen = new HashSet<>();
            for (int j = i + 1; j < nums.length; j++) {
                int complement = -nums[i] - nums[j];
                if (seen.contains(complement)) {
                    List<Integer> triplet = Arrays.asList(nums[i], complement, nums[j]);
                    Collections.sort(triplet);
                    resultSet.add(triplet);
                }
                seen.add(nums[j]);
            }
        }

        return new ArrayList<>(resultSet);
    }
}
```

**为什么不推荐？**
- 去重逻辑不直观，需要额外的 Set
- 空间复杂度 O(n)
- 代码可读性差
